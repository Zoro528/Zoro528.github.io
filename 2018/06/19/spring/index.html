后台管理系统总结

ApplicationContext对象用于注入对象

从xml配置文件中通过context.getBean("name")注入对象

spring两种注入方式

setter注入和构造函数注入，setter注入可以添加@require注解来强制要求注入对象，否则在容器初始化时报错

mybatis-generator

配置xml文件并在控制台运行

shiro

AuthenticationInfo代表了用户的角色信息集合，AuthorizationInfo代表了角色的权限信息集合

text格式mybatis-generator处理

mapper中的条件判断与模糊查询

多项遍历取值或删除

mapper中的大于&gt;和小于&lt;以及分页 limit 3，5
sql更新某一字段：UPDATE sh_hive_humidity set apiary_id = 22 WHERE id >1000 and id <2000

git版本管理

通过git pull与远端同步、用git commit控制版本、用git reset -soft -mixed -hard控制版本回退、用git push同步到远程git仓库

hibernate

hibernate查询创建基础

hibernate通过HQL来完成数据库操作并将其映射到实体类上

也可通过原生sql完成数据库操作

hibernate缓存机制

一级缓存：一种强制缓存，默认使用一级缓存；二级缓存：

html访问本地资源file:///e:/code/js/new.html
页数取整（假设每页10条数据）int pages = (count+9)/10;

正向代理与反向代理

a,b,c三个人，正向代理是a通过b向C借钱，a知道c的存在 。反向代理是a向b借钱，b又向C借，a不知道c的存在
正向代理：翻墙，反向代理：Nginx

HashMap无序输出

用LinkedHashMap代替HashMap即可实现有序输入（例如：excel导出中的Map表头的定义）。

无Spring时MybatisMapper的注入

由SqlSessionFactoryBuilder 创建SqlSessionFactory ，然后由SqlSessionFactory创建SqlSession 实例。
